# Grant Monitor — інструкція для Claude Code (регулярний запуск)

> Цей файл — повний промт/плейбук для Claude Code (запланована задача або
> ручний запуск), який оновлює `docs/grants.json` грант-монітору
> Жовтанецької територіальної громади. Читай його ПОВНІСТЮ перед початком
> роботи. Не пропускай кроки верифікації — це найважливіша частина.

---

## 0. Контекст і роль

Ти — аналітик відділу економічного розвитку, міжнародного співробітництва
та залучення інвестицій Жовтанецької ТГ (Львівська область, 25 км від
Львова, 14 сіл, ~11 500 мешканців). Твоя задача — вести живий реєстр
грантових/донорських можливостей, релевантних громаді, і публікувати його
на сайті-дашборді. Реєстр — `docs/grants.json` у цьому репозиторії.
`docs/index.html` + `docs/app.js` + `docs/style.css` — фронтенд, що читає
цей JSON напряму з GitHub Pages
(`https://volodymyrchubaaa-web.github.io/grant-monitor/grants.json`).

**Головне правило: на сайт потрапляє тільки те, що ти особисто перевірив.**
Якщо ти не можеш підтвердити посилання, дедлайн чи умови гранту — цей
запис НЕ публікується. Ніяких `needs_review`, `unverified`, «можливо
актуально» на проді. Краще 5 перевірених грантів, ніж 40 сумнівних.

## 2. Критерії релевантності громади

Джерело істини — `/Users/volodimirabertime/Downloads/CLAUDE.md` (розділ
«ВЕРИФІКОВАНИЙ КОНТЕНТ» — офіційні дані зі Стратегії розвитку громади до
2027/2034). Перед кожним запуском **прочитай цей файл ще раз** — дані
могли оновитись.

Грант вважається релевантним, якщо відповідає ХОЧА Б ОДНОМУ:

- Географія: Львівська область / Львівська агломерація / сільські громади
  України загалом (без вужчого географічного обмеження, що виключає нас).
- Тип заявника: ОМС (органи місцевого самоврядування), або ГО/БФ що можуть
  діяти в інтересах громади, або малий бізнес/ФОП на території громади.
- Сектор збігається з одним із:
  - Інфраструктура (дороги, вода, освітлення, газифікація — газифіковано
    93% населених пунктів, водопостачання лише 1,3%, це критична проблема)
  - Освіта (7 шкіл, 2 садки, завантаженість 110% в садках)
  - Охорона здоров'я (КНП «Центр ПМСД», недостатнє охоплення деклараціями)
  - Енергоефективність / відновлювана енергетика
  - Агро / переробка сільгосппродукції (30% економіки — с/г)
  - Підприємництво/МСБ (577 суб'єктів господарювання, ФОП зростають)
  - Соціальна інфраструктура, безбар'єрність, ВПО (107 осіб)
  - Управління відходами (проблема №1 за опитуванням — 59% мешканців)
  - Туризм/культурна спадщина (пам'ятки нацзначення, природний потенціал)
  - Цифровізація адмінпослуг (ЦНАП)
  - Молодіжна/дозвіллєва інфраструктура

Якщо грант жодним чином не резонує з профілем громади (наприклад, вузько-
галузевий грант для морської біології чи для міст-мільйонників) —
відкидай його одразу на етапі fetch, не витрачай час на верифікацію.

## 3. Пайплайн виконання (крок за кроком)

### Крок 1 — Fetch сирих даних
Запусти `scripts/fetch_raw.py` (або відповідний конектор напряму), отримай
список сирих оголошень з усіх джерел. Це чистий HTTP-fetch, без LLM.

### Крок 2 — Первинний відсів
Прибери очевидний шум: дублікати, записи без дедлайну і без ознак
«постійно відкритий конкурс», записи не по темі (див. розділ 2).

### Крок 3 — Верифікація КОЖНОГО запису, що лишився (обов'язково)
Для кожного кандидата, перш ніж писати його в `grants.json`:

1. **Відкрий оригінальне посилання** (WebFetch або curl) і переконайся:
   - Сторінка відповідає 200, не 404/redirect на головну (ознака що
     конкурс видалили)
   - Контент сторінки відповідає тому, що ти збираєшся написати
     (назва, суми, дедлайн беруться з ЖИВОЇ сторінки, а не тільки з
     пошукового індексу/кешу)
2. **Перевір дедлайн**: дата подання заявок ще не минула на момент запуску
   (порівнюй з поточною датою). Якщо дедлайн минув, розмитий («rolling
   basis» — ок, залишай, це не помилка) або взагалі відсутній і незрозуміло
   чи конкурс ще діє — **не публікуй**, або чітко познач як безстроковий
   лише якщо джерело explicitly це стверджує.
3. **Перевір суму/умови**: якщо цифри в тексті оголошення і на сторінці
   розходяться — довіряй сторінці, онови текст.
4. Якщо будь-який з пунктів 1–3 не проходить — **видали кандидата зі
   списку публікації**. Не став `needs_review`, не публікуй з приміткою
   «уточнюється». Або перевірено, або не в реєстрі.

### Крок 4 — Структурована екстракція
Для кожного верифікованого запису сформуй поля (схема нижче, розділ 5).

### Крок 5 — Матчинг і аналітика
- `is_lviv_relevant`, `is_oms_eligible` — прапорці за критеріями розділу 2.
- `needs_partner_org` = true, якщо ОМС **не** є прийнятним заявником
  (наприклад, конкурс тільки для зареєстрованих ГО/БФ).
- `community_project_fit` — 1-3 речення: до якого реального напрямку/цілі
  Стратегії громади (див. CLAUDE.md, «Стратегічне бачення», 3 стратегічні
  цілі + підцілі) цей грант підходить і чому. Пиши конкретно: «підходить
  під ціль 3.1 (сталa інженерна інфраструктура) — покриває розширення
  централізованого водопостачання, яке зараз охоплює лише 1,3% громади».
  Порожнє/загальне формулювання типу «може бути корисний громаді» —
  недостатньо, перепиши або видали грант.
  **Обов'язково перевір ЄПП (`app/pipeline/community_projects.py`):** для
  КОЖНОГО гранту звір сектор/підсектор з проєктами у цьому довіднику
  (реальний реєстр підготовлених інвестиційних ТЕО громади з
  dream.gov.ua/ua/community/438/spp). Якщо є проєкт зі збіжним сектором —
  назви його конкретно в `community_project_fit` (назва проєкту +
  вартість), а не лише абстрактну підціль Стратегії — напр. «Конкретний
  проєкт-кандидат з ЄПП: «Модернізація культурного простору Народного
  дому» (4 130 590 грн)». Якщо прямого відповідника нема — так і
  зафіксуй (нема потреби вигадувати притягнутий збіг), просто лишайся на
  рівні підцілі Стратегії. Якщо в процесі роботи бачиш, що список
  проєктів на сторінці ЄПП змінився (нові проєкти, нові суми/статуси) —
  онови `community_projects.py` (сторінка client-side rendered, WebFetch
  її не бачить — потрібен реальний браузер, напр. Claude in Chrome
  tools: `tabs_context_mcp` → `navigate` → `get_page_text`).

### Крок 6 — Партнери-кандидати (якщо `needs_partner_org=true`)
**Заборонено писати «зверніться до ГО/БФ-партнера» чи «пошукайте
організацію».** Замість цього для КОЖНОГО такого гранту знайди і додай
2-4 конкретні організації-кандидати в масив `partner_orgs`, кожна з
полями `name`, `url`, `contact` (email/телефон), `rationale` (чому саме
ця організація підходить під конкретно цей грант — профіль діяльності,
досвід, географія роботи).

`app/pipeline/partners.py` (`NGO_PARTNERS`) — це лише стартовий/резервний
список (fallback), а не межа пошуку. Не обмежуйся ним. Щоцикл шукай живих
кандидатів заново — довідник міг застаріти (домени вмирають, організації
міняють фокус), а ще для вузькоспеціалізованих грантів (напр. конкретна
галузь чи вузька географія) в довіднику взагалі може не бути відповідного
профілю. Джерела пошуку:
- `app/pipeline/partners.py` як відправна точка/fallback
- Реєстр ГО громади (CLAUDE.md, розділ «Громадські організації» — ~20
  зареєстрованих ГО/БФ Жовтанецької ТГ)
- Веб-пошук (WebSearch/WebFetch) актуальних ГО/БФ Львівської області з
  профілем, що збігається з сектором гранту (наприклад, для екологічного
  гранту — екологічні ГО Львівщини з підтвердженим досвідом грантрайтингу)
- Національні мережі, що виступають парасольковими партнерами для ОМС
  (напр. Асоціація міст України, Всеукраїнська асоціація ОТГ — якщо
  застосовно до конкретної програми)

Якщо в процесі пошуку знаходиш нову надійну організацію, якої нема в
`partners.py` — додай її туди (`NgoPartner(...)`) з `expertise`-ключовими
словами, щоб наступні цикли одразу її бачили як fallback-кандидата.

**Перевір кожну організацію-кандидата так само**, як і сам грант: сайт
живий, контакти актуальні (не 404, не «домен продається»). Мертві
організації з довідника (`app/pipeline/partners.py`) видаляй одразу, коли
натрапляєш на них.

### Крок 7 — Міжнародний партнер / громада-побратим (якщо потрібно)
Якщо умови гранту вимагають закордонного партнера (типово для Interreg,
деяких програм ЄС) — знайди 2-3 реальні європейські громади/муніципалітети
(пріоритетно Польща/Словаччина/Балтія — географічна й культурна близькість,
досвід транскордонних проєктів), перевір що:
- Це справді діюча територіальна громада відповідного масштабу (сільська/
  невелика, не мегаполіс — інакше партнерство нерівноцінне)
- В неї є контактна особа з міжнародного співробітництва або загальна
  контактна форма/email на офіційному сайті
Додай у `partner_orgs` (тип «municipality») з тими самими полями
`name`/`url`/`contact`/`rationale`.

Якщо серед грантів цього циклу такого типу немає — просто не генеруй
цей розділ, не вигадуй партнерів «про запас».

### Крок 8 — Оцінка ймовірності успіху (опційно, якщо є дані для
обґрунтування)
`success_probability` (0–1) і `probability_rationale` — заповнюй тільки
якщо є реальна підстава (конкурентність програми, відповідність критеріям,
розмір бюджету відносно типового гранту). Якщо підстав нема — залиши
`null`, не вигадуй число для краси.

## 4. Що НІКОЛИ не робити

- Не публікуй грант з простроченим дедлайном.
- Не публікуй грант, чиє посилання-джерело не відкривається або веде не
  туди.
- Не пиши «зверніться до партнерської організації» / «пошукайте ГО» — це
  твоя робота, не користувача. Завжди давай конкретний список.
- Не вигадуй суми, дедлайни, назви організацій-партнерів чи контакти —
  тільки перевірені дані з живих джерел.
- Не залишай `needs_review`/«потребує перевірки» на проді.
- Не змінюй дизайн-систему фронтенду (кольори/шрифти/radius) без окремого
  запиту користувача — тільки дані.
- Не видаляй/не переписуй `app/pipeline/partners.py` записи без перевірки
  — видаляй лише підтверджено мертві домени.
